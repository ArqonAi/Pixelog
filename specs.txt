     * Empty Go project directory with only documentation files
     * Go 1.25.1 installed and ready
     * FFmpeg 7.1.1 available with necessary codecs (libx264, yuv420p)
     * Clear project vision: Pixelog - a "SQLite-meets-YouTube for LLM memories"

   Project Concept: Pixelog stores diverse knowledge sources (text, images, audio,
   micro-videos) as compressed, encrypted video files using QR code streams that
   are portable, streamable, and semantically searchable.

   ### Technical Specifications:

   #### QR Code Settings:

     * Resolution: 512x512 pixels minimum for readability
     * Error Correction: Medium level (Level M, ~15% recovery)
     * Data Capacity: ~2,953 bytes per QR code (alphanumeric)
     * Chunk Size: 2,800 bytes to leave room for metadata

   #### Video Settings:

     * Codec: libx264 with yuv420p pixel format
     * Frame Rate: 0.5-1 FPS (configurable)
     * Audio: Silent track using anullsrc
     * Container: MP4 with fast-start enabled
     * Compression: CRF 23 for good quality/size balance

   #### File Processing:

     * Binary Files: Base64 encoding with MIME type detection
     * Text Files: UTF-8 encoding with chunk boundaries on word/sentence breaks
     * Metadata: JSON format embedded in QR codes or separate metadata track
